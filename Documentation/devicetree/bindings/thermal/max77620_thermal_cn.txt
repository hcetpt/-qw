Maxim Semiconductor 的 MAX77620 电源管理 IC 的热驱动程序。MAX77620 支持在其芯片温度超过 120°C 和 140°C 时触发报警中断。这些阈值温度是不可配置的。该设备除了指示温度是否高于或低于阈值水平外，不提供实际的芯片温度。

**必需属性：**
-------------------
`#thermal-sensor-cells`: 更多详细信息，请参考
            `<devicetree/bindings/thermal/thermal-sensor.yaml>`
            值必须为 0

更多信息请参阅通用热设备树绑定文档
`<devicetree/bindings/thermal/thermal*.yaml>`

关于 MAX77620 的多功能设备（mfd）设备树绑定文档，请参考
`<devicetree/bindings/mfd/max77620.txt>`

**示例：**
--------

```yaml
#include <dt-bindings/mfd/max77620.h>
#include <dt-bindings/thermal/thermal.h>

...

i2c@7000d000 {
    spmic: max77620@3c {
        compatible = "maxim,max77620";
        ::::
        #thermal-sensor-cells = <0>;
        :::
    };
};

cool_dev: cool-dev {
    compatible = "cooling-dev";
    #cooling-cells = <2>;
};

thermal-zones {
    PMIC-Die {
        polling-delay = <0>;
        polling-delay-passive = <0>;
        thermal-sensors = <&spmic>;

        trips {
            pmic_die_warn_temp_thresh: hot-die {
                temperature = <120000>; // 单位为微度 (μ°C)
                type = "hot";
                hysteresis = <0>;
            };

            pmic_die_cirt_temp_thresh: critical-die {
                temperature = <140000>; // 单位为微度 (μ°C)
                type = "critical";
                hysteresis = <0>;
            };
        };

        cooling-maps {
            map0 {
                trip = <&pmic_die_warn_temp_thresh>;
                cooling-device = <&cool_dev THERMAL_NO_LIMIT
                                  THERMAL_NO_LIMIT>;
                contribution = <100>;
            };
        };
    };
};
```

在这个示例中，我们定义了一个 MAX77620 芯片，它通过 I2C 总线连接，并指定了热传感器单元的数量为 0。接着定义了一个冷却设备，并且在一个名为 `thermal-zones` 的节点中定义了热区域。该热区域定义了两个阈值事件：当温度达到 120°C 时发出警告（类型为“hot”），当温度达到 140°C 时发出严重警告（类型为“critical”）。此外，还定义了一个冷却映射，当温度超过警告阈值时，启用冷却设备进行降温。
