* 上传了 QE 固件

      如果有新的固件被上传到 QE（通常由启动加载器完成），则应向 QE 节点添加一个名为“固件”的子节点。此节点提供了设备驱动程序可能需要的已上传固件的相关信息。
必需属性：
      - id：固件的字符串名称。这个名称取自已上传固件中的 `qe_firmware` 结构的 `id` 成员
设备驱动程序可以搜索该字符串来判断它们所需的固件是否已经存在
- extended-modes：扩展模式位字段，从固件二进制文件中获取。它是一个表示为两个 32 位数字数组的 64 位数字
- virtual-traps：虚拟陷阱，从固件二进制文件中获取。它是一个包含八个 32 位数字的数组
示例：
	firmware {
		id = "Soft-UART";
		extended-modes = <0 0>;
		virtual-traps = <0 0 0 0 0 0 0 0>;
	};
