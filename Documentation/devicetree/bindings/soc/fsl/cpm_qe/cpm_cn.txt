* 摩托罗拉通信处理器模块 (Freescale Communications Processor Module)

注释：这是一个临时绑定，随着支持的设备增加可能会有轻微的变化。特别是QE绑定还不完整。
* 根CPM节点

属性：
- compatible: "fsl,cpm1"、"fsl,cpm2" 或 "fsl,qe"
- reg: 一个从CPCR开始的48字节区域
示例：
     cpm@119c0 {
	#address-cells = <1>;
	#size-cells = <1>;
	#interrupt-cells = <2>;
	compatible = "fsl,mpc8272-cpm", "fsl,cpm2";
	reg = <119c0 30>;
     }

* 对多个CPM/QE设备通用的属性

- fsl,cpm-command: 这个值与操作码和命令标志进行或运算，以指定CPM命令所作用的设备
- fsl,cpm-brg: 表明该设备关联的是哪个波特率发生器（BRG）。如果未设置，则应动态分配一个未使用的BRG；如果为零，则表示该设备使用外部时钟而不是BRG
- reg: 除非另有说明，第一个资源代表scc/fcc/ucc寄存器，第二个资源代表设备的参数RAM区域（如果有）
* 多用户RAM (MURAM)

多用户/双端口RAM作为CPM节点下的总线表示。
必须根据以下限制来设置范围：

- 子节点的reg节点必须是从所有MURAM起始地址的偏移量，即使用户数据区不是从零开始
- 如果使用了多个范围条目，则父节点地址与子节点地址之间的差值在所有条目中都必须相同，以便可以通过单一映射覆盖它们，同时保持通过指针减法确定CPM侧偏移的能力。建议不要使用多个范围条目
- 即使没有包含它的reg资源，子节点地址为零也必须可被转换
必须存在一个子节点 "数据" 节点，与 "fsl,cpm-muram-data" 兼容，用于指示操作系统可用于任意目的的那部分 MURAM（内存）。该数据节点可以有任意数量的 `reg` 资源，所有这些资源都为可分配的 MURAM 池做出了贡献。

以下是一个基于 mpc8272 的示例：
``` 
muram@0 {
	#address-cells = <1>;
	#size-cells = <1>;
	ranges = <0 0 10000>;

	data@0 {
		compatible = "fsl,cpm-muram-data";
		reg = <0 2000 9800 800>;
	};
};
```
在此示例中：

- `muram@0` 表示 MURAM 的根节点。
- `#address-cells` 和 `#size-cells` 定义了地址和大小单元的数量。
- `ranges` 定义了地址空间的映射。
- `data@0` 是与 "fsl,cpm-muram-data" 兼容的数据节点，它定义了操作系统可以用于任意用途的 MURAM 部分。
- `reg = <0 2000 9800 800>` 表示从地址 0x2000 开始，长度为 0x800 的 MURAM 区域，这部分 MURAM 可供操作系统使用。
