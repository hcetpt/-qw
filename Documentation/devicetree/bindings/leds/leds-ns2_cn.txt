在Network Space v2（及其父型号）上找到的双GPIO LED绑定。

必需属性：
- `compatible`: `"lacie,ns2-leds"`
每个LED都作为`ns2-leds`设备的一个子节点表示。

必需的子节点属性：
- `cmd-gpio`: 命令LED的GPIO。请参阅OF设备树GPIO规范。
- `slow-gpio`: 慢速LED的GPIO。请参阅OF设备树GPIO规范。
- `modes-map`: LED模式（关闭、打开或SATA活动闪烁）与相应的`cmd-gpio/slow-gpio`值之间的映射。应给出所有GPIO值组合，以避免在驱动程序探测时出现未知模式。

可选子节点属性：
- `label`: 此LED的名称。如果省略，则从节点名称中获取标签。
- `linux,default-trigger`: 分配给LED的触发器

示例：

```dts
#include <dt-bindings/leds/leds-ns2.h>

ns2-leds {
	compatible = "lacie,ns2-leds";

	blue-sata {
		label = "ns2:blue:sata";
		slow-gpio = <&gpio0 29 0>;
		cmd-gpio = <&gpio0 30 0>;
		modes-map = <NS_V2_LED_OFF  0 1
			     NS_V2_LED_ON   1 0
			     NS_V2_LED_ON   0 0
			     NS_V2_LED_SATA 1 1>;
	};
};
```

这段配置描述了一个针对Network Space v2的双GPIO LED的Device Tree片段。其中定义了一个名为`blue-sata`的LED，它具有命令和慢速两种GPIO设置，并且定义了不同模式下的GPIO状态。
