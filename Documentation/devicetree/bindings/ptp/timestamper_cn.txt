来自MII总线窥探设备的时间戳

此绑定支持非PHY设备，这些设备窥探MII总线并提供时间戳。与PHY时间戳驱动程序（可以直接将其接口连接到PHY实例）不同，独立的MII时间戳驱动程序使用此绑定来指定窥探设备与特定网络接口之间的连接。非PHY MII时间戳驱动程序通常通过另一个总线（如I2C、SPI、UART或通过内存映射的外围设备）与控制接口通信。此控制器设备与一个或多个时间戳通道相关联，每个通道都窥探MII总线。

“timestamper”属性位于phy节点中，并将控制器设备的时间戳通道链接到该phy的MII总线。

示例：

	tstamper: timestamper@10000000 {
		compatible = "ines,ptp-ctrl";
		reg = <0x10000000 0x80>;
	};

	ethernet@20000000 {
		mdio {
			ethernet-phy@1 {
				timestamper = <&tstamper 0>;
			};
		};
	};

	ethernet@30000000 {
		mdio {
			ethernet-phy@2 {
				timestamper = <&tstamper 1>;
			};
		};
	};

在此示例中，连接到phy@1的MII总线上的时间戳将出现在时间戳通道0（零）上，而phy@2上的时间戳将出现在通道1上。
