```text
2. 支持的适配器
=====================

网络驱动程序 'skfp' 支持以下网络适配器：
SysKonnect 适配器：

  - SK-5521 (SK-NET FDDI-UP)
  - SK-5522 (SK-NET FDDI-UP DAS)
  - SK-5541 (SK-NET FDDI-FP)
  - SK-5543 (SK-NET FDDI-LP)
  - SK-5544 (SK-NET FDDI-LP DAS)
  - SK-5821 (SK-NET FDDI-UP64)
  - SK-5822 (SK-NET FDDI-UP64 DAS)
  - SK-5841 (SK-NET FDDI-FP64)
  - SK-5843 (SK-NET FDDI-LP64)
  - SK-5844 (SK-NET FDDI-LP64 DAS)

康柏适配器（未经测试）：

  - Netelligent 100 FDDI DAS 纤维 SC
  - Netelligent 100 FDDI SAS 纤维 SC
  - Netelligent 100 FDDI DAS UTP
  - Netelligent 100 FDDI SAS UTP
  - Netelligent 100 FDDI SAS 纤维 MIC

4. 故障排除
==================

如果在安装过程中遇到问题，请检查以下内容：

问题：
	  驱动程序找不到 FDDI 适配器
```
原因：
请检查 `/proc/pci` 文件中是否存在以下条目：

     ‘FDDI 网络控制器: SysKonnect SK-FDDI-PCI ...’

如果存在该条目，则表示系统已经识别到了 FDDI 适配器，应该可以使用。
如果此条目不存在或者文件 `/proc/pci` 不存在，则您可能遇到了硬件问题或内核中未启用 PCI 支持。
可以通过 SysKonnect 网站提供的诊断程序来检查适配器：

      www.syskonnect.de

一些 COMPAQ 计算机在 Linux 下存在 PCI 的问题。这个问题在“PCI 如何”文档中有描述（包含在某些发行版中或可从网站获得，例如在 'www.linux.org'），目前没有可用的解决方法。
问题：
您希望将计算机用作多个 IP 子网之间的路由器（使用多个适配器），但无法访问其他子网中的计算机。
原因：
要么是路由器的内核没有配置为 IP 转发功能，要么至少有一台计算机的路由表和网关配置存在问题。
如果您遇到的问题没有列在这里，请联系我们的技术支持获取帮助。
您可以发送电子邮件至：linux@syskonnect.de

在联系我们的技术支持时，
请确保提供以下信息：

- 系统制造商和型号
- 您系统中的板卡
- 发行版
- 内核版本

5. 适配器 LED 的功能
===============================

FDDI 网络适配器上的 LED 功能在 SMT 版本 v2.82 中进行了更改。
使用此新版本的 SMT，黄色 LED 作为环路运行指示器。亮起的黄色 LED 表示环路处于停机状态。适配器上的绿色 LED 现在作为连接指示器，其中亮起的绿色 LED 表示相应的端口具有物理连接。
在 SMT 版本 v2.82 之前的版本中，如果黄色 LED 关闭而绿色 LED 显示适配器的连接状态，则表示环路已启动。在环路停机期间，绿色 LED 关闭且黄色 LED 亮起。
所有实现都表明如果没有加载驱动程序则所有 LED 都会关闭。
