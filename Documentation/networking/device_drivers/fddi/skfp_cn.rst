1. 概述
===========

本 README 解释了如何使用驱动程序 'skfp' 在您的网络适配器上运行 Linux。
第 2 章：包含所有受此驱动程序支持的网络适配器列表。
第 3 章：提供一些一般信息。
第 4 章：描述常见问题及解决方案。
第 5 章：展示适配器 LED 功能的变化。
第 6 章：开发历史。

2. 支持的适配器
=====================

网络驱动程序 'skfp' 支持以下网络适配器：
SysKonnect 适配器：

  - SK-5521 (SK-NET FDDI-UP)
  - SK-5522 (SK-NET FDDI-UP DAS)
  - SK-5541 (SK-NET FDDI-FP)
  - SK-5543 (SK-NET FDDI-LP)
  - SK-5544 (SK-NET FDDI-LP DAS)
  - SK-5821 (SK-NET FDDI-UP64)
  - SK-5822 (SK-NET FDDI-UP64 DAS)
  - SK-5841 (SK-NET FDDI-FP64)
  - SK-5843 (SK-NET FDDI-LP64)
  - SK-5844 (SK-NET FDDI-LP64 DAS)

Compaq 适配器（未经测试）：

  - Netelligent 100 FDDI DAS Fibre SC
  - Netelligent 100 FDDI SAS Fibre SC
  - Netelligent 100 FDDI DAS UTP
  - Netelligent 100 FDDI SAS UTP
  - Netelligent 100 FDDI SAS Fibre MIC

3. 一般信息
======================

从 v2.01 版本开始，该驱动程序已集成到 Linux 内核源代码中。
因此，安装与其他内核支持的适配器相同。
请参阅您的发行版手册以获取网络适配器的安装指南。
这使我的工作变得轻松多了 :-)

4. 故障排除
==================

如果您在安装过程中遇到问题，请检查以下事项：

问题：
      FDDI 适配器无法被驱动程序找到
原因：
请检查 `/proc/pci` 目录下的以下条目：

     'FDDI 网络控制器: SysKonnect SK-FDDI-PCI ...'

如果存在此条目，则表示系统已经检测到了 FDDI 适配器，应该可以使用。
如果此条目不存在或 `/proc/pci` 文件不存在，则可能存在硬件问题或内核中未启用 PCI 支持。
可以通过 SysKonnect 网站提供的诊断程序来检查适配器：

      www.syskonnect.de

某些 COMPAQ 计算机在 Linux 下存在 PCI 问题。这在“PCI 如何”文档中有描述（包含在某些发行版中或可以从 www 获取，例如在 'www.linux.org'），并且没有可用的解决方法。

问题：
您希望将计算机用作多个 IP 子网之间的路由器（使用多个适配器），但无法访问其他子网中的计算机。
原因：
要么路由器的内核未配置为 IP 转发，要么至少有一台计算机的路由表和网关配置存在问题。
如果您遇到的问题不在这里列出，请联系我们的技术支持寻求帮助。
您可以发送电子邮件至：linux@syskonnect.de

在联系我们的技术支持时，
请确保提供以下信息：

- 系统制造商和型号
- 您系统中的板卡
- 发行版
- 内核版本

5. 适配器 LED 的功能
===============================

FDDI 网络适配器上的 LED 功能在 SMT 版本 v2.82 中有所更改。使用这个新的 SMT 版本，黄色 LED 作为环路运行指示器。活动的黄色 LED 表示环路已关闭。适配器上的绿色 LED 作为链路指示器，其中活动的绿色 LED 表示相应的端口有物理连接。
在 SMT 版本 v2.82 之前的版本中，如果黄色 LED 关闭而绿色 LED 显示适配器的连接状态，则表示环路正常。在环路关闭期间，绿色 LED 关闭，黄色 LED 开启。
所有实现都表明，如果所有 LED 都关闭，则表示未加载驱动程序。
