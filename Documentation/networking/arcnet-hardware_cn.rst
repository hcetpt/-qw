ARCnet 简介
======================

ARCnet 是一种网络类型，其工作方式与流行的以太网类似但也有许多重要的不同之处。首先，您可以获得至少两种速度的 ARCnet 卡：2.5 Mbps（比以太网慢）和 100 Mbps（比标准以太网快）。实际上还有其他类型，但它们较为罕见。据我所知，不同的硬件类型之间并不兼容，因此您不能将 100 Mbps 卡连接到 2.5 Mbps 卡等。据我了解，我的驱动程序可以支持 100 Mbps 卡，但我自己无法验证这一点，因为我只有 2.5 Mbps 类型的卡。它可能无法充分利用您的 100 Mbps 卡。别抱怨了。

您也不能将 ARCnet 卡连接到任何类型的以太网卡并期望它能正常工作。
ARCnet 有两种“类型”——星形拓扑和总线拓扑。这指的是卡片应该如何连接在一起。根据大多数可用文档，您只能将星形拓扑卡连接到星形拓扑卡，将总线拓扑卡连接到总线拓扑卡。这说得通，对吧？嗯，这不完全正确；请参见下面的“布线”。

一旦您克服了这些小障碍，ARCnet 实际上是一个设计得相当好的标准。它使用所谓的“改进令牌传递”，这使得它与所谓的“令牌环”卡完全不兼容，但这种传递方式使传输比以太网更可靠。事实上，ARCnet 可以保证数据包安全到达目的地，并且即使不可能正确传送（例如，因为电缆断裂或目标计算机不存在），它也会向发送方报告情况。由于令牌的精心定义行为，它总会在最大限定时间内绕“环”传递一次。这使其适用于实时网络。此外，所有已知的 ARCnet 卡都有一个（几乎）相同的编程接口。这意味着用一个 ARCnet 驱动程序就可以支持任何卡，而以太网每个制造商使用的编程接口有时完全不同，导致存在很多不同的、有时非常相似的以太网驱动程序。当然，始终使用相同的编程接口也意味着当出现高性能硬件设施如 PCI 总线主控 DMA 时，很难利用它们。我们就不讨论这个问题了。
然而，使 ARCnet 卡难以编程的一点是其数据包大小的限制；标准 ARCnet 只能发送长度最多为 508 字节的数据包。这小于互联网的“最低要求”576 字节，更不用说以太网的 MTU 1500 字节了。为了弥补这一点，RFC1201 定义了一个额外的封装级别，我称之为“数据包拆分”，允许“虚拟数据包”的大小扩展到每个 64K，尽管通常会将其保持在以太网风格的 1500 字节左右。有关 ARCnet 网络优缺点（主要是优点）的更多信息，您可以尝试访问“ARCnet 贸易协会”的网页：

	http://www.arcnet.com

ARCnet（经典的 2.5 Mbps 版本）可以通过两种不同类型的电缆连接：同轴电缆和双绞线。其他的 ARCnet 类型网络（100 Mbps TCNS 和 320 kbps - 32 Mbps ARCnet Plus）使用其他类型的电缆（Type1、光纤、C1、C4、C5）。
对于同轴网络，您“应该”使用 93 欧姆 RG-62 电缆。但其他电缆也能很好地工作，因为 ARCnet 是一个非常稳定的网络。我个人使用的是 75 欧姆电视天线电缆。
同轴电缆卡以两种不同变体的形式发货：用于总线（BUS）和星型（STAR）网络拓扑。它们在很大程度上是相同的，唯一的区别在于安装的混合芯片。总线（BUS）卡使用高阻抗输出，而星型（STAR）则使用低阻抗。低阻抗卡（STAR）在电气特性上等同于带有终端器安装的高阻抗卡。

通常，ARCnet网络由星型（STAR）卡和集线器构建而成。有两种类型的集线器 - 主动和被动。被动集线器是带有四个BNC连接器的小盒子，内部含有四个47欧姆电阻：

```
	   |         | wires
	   R         + junction
	-R-+-R-      R 47欧姆电阻
	   R
	   |
```

屏蔽层相互连接。主动集线器要复杂得多；它们需要供电，并且包含电子设备来放大信号并将其发送到网络的其他部分。它们通常有八个连接器。主动集线器有两种变体 - 傻瓜式和智能式。傻瓜式仅放大信号，但智能式解码所有通过的数字包并重新编码。如果你在网络中有多个集线器，这会好得多，因为过多的傻瓜式主动集线器可能会降低信号质量。

现在来说说电缆连接。你可以这样连接：

1. 卡到卡。这是创建两台计算机网络最简单的方式。
2. 卡到被动集线器。记住，集线器上未使用的连接器必须用93欧姆（或其他，如果没有正确的）终端器正确终止。
3. 卡到主动集线器。这里不需要终止未使用的连接器，除非出于某种美观考虑。但是，在任何两台计算机之间不能有多于十一个主动集线器。当然，这并不限制网络中主动集线器的数量。
4. 一个主动集线器到另一个
5. 一个主动集线器到被动集线器
记住，你不能将两个被动集线器连接在一起。这种连接带来的功率损失太高，以至于网络无法可靠运行。

典型的ARCnet网络示例：

```
	   R                     S - 星型（STAR）类型卡
    S------H--------A-------S    R - 终端器
	   |        |            H - 集线器
	   |        |            A - 主动集线器
	   |   S----H----S
	   S        |
		    |
		    S
```

总线（BUS）拓扑与以太网使用的非常相似。唯一的区别在于电缆和终端器：它们应该是93欧姆。以太网使用50欧姆阻抗。你使用T型连接器将计算机放置在单根电缆线上，即总线。你必须在电缆的两端放置终端器。典型的总线（BUS）ARCnet网络如下：

```
    RT----T------T------T------T------TR
     B    B      B      B      B      B

  B - 总线（BUS）类型卡
  R - 终端器
  T - T型连接器
```

但这还不是全部！这两种类型可以相互连接。根据官方文档，唯一允许的连接方式是使用主动集线器：

```
	 A------T------T------TR
	 |      B      B      B
     S---H---S
	 |
	 S
```

官方文档还指出，你可以在总线（BUS）网络的两端使用星型（STAR）卡代替总线（BUS）卡和终端器：

```
     S------T------T------S
	    B      B
```

但根据我自己的实验，你可以在星型（STAR）拓扑网络中的电缆中间挂接总线（BUS）类型卡。更进一步，如果你使用终端器，你可以用总线（BUS）卡替换任何星型（STAR）卡。然后，你可以构建满足你所有需求的非常复杂的网络！例如：

```
				  S
				  |
	   RT------T-------T------H------S
	    B      B       B      |
				  |       R
    S------A------T-------T-------A-------H------TR
	   |      B       B       |       |      B
	   |   S                 BT       |
	   |   |                  |  S----A-----S
    S------H---A----S             |       |
	   |   |      S------T----H---S   |
	   S   S             B    R       S
```

使用双绞线（Twisted Pair）布线方案则完全不同。每个TP卡有两个RJ（电话线风格）连接器。卡片通过连接每两张相邻卡片的电缆串连起来。两端用RJ 93欧姆终端器终止，这些终端器插入链尾卡片空余的连接器中。例如：

```
	  ___________   ___________
      _R_|_         _|_|_         _|_R_
     |     |       |     |       |     |
     |Card |       |Card |       |Card |
     |_____|       |_____|       |_____|
```

双绞线（TP）拓扑也有集线器。使用它们没有困难；你只需将TP链的一端或两端连接到集线器即可。这样，你可以几乎创建任何网络配置。

任何两台计算机之间的最大集线器数量为11个的限制也适用于此。例如：

```
    RP-------P--------P--------H-----P------P-----PR
			       |
      RP-----H--------P--------H-----P------PR
	     |                 |
	     PR                PR

    R - RJ 终端器
    P - TP 卡
    H - TP 集线器
```

就像任何网络一样，ARCnet也有有限的电缆长度。以下是在两个活动端点（活动端点可以是主动集线器或星型（STAR）卡）之间的最大电缆长度。
下面是给定内容的中文翻译：

RG-62       93 欧姆，最长可达 650 米  
RG-59/U     75 欧姆，最长可达 457 米  
RG-11/U     75 欧姆，最长可达 533 米  
IBM Type 1  150 欧姆，最长可达 200 米  
IBM Type 3  100 欧姆，最长可达 100 米  

连接到被动集线器的所有电缆的最大长度对于 RG-62 电缆限制为 65 米；其他类型的电缆则更短。你可以看到，在大型网络中使用被动集线器是一个糟糕的想法。单个“总线主干”（Bus Trunk）的最大长度对于 RG-62 约为 300 米。网络中两个最远点之间的最大距离限制在 3000 米。两个卡/集线器之间 TP 电缆的最大长度为 650 米。

设置跳线
所有 ARCnet 卡应该有总共四到五个不同的设置：

- 输入/输出地址：这是你的 ARCnet 卡所在的“端口”。Linux ARCnet 驱动程序中探测到的值只从 0x200 到 0x3F0。（如果你的卡有额外的地址，请告知我。）这不应该与系统上的任何其他设备相同。根据我从 Novell 获得的一份文档，MS Windows 更倾向于 0x300 或以上的值，否则会吞噬我的系统（至少）的网络连接。我的猜测是，如果您的卡位于 0x2E0，那么在 0x2E8 探测串行端口可能会重置卡，并可能彻底搞乱事情。
- Avery 的首选：0x300
- 中断请求（IRQ）：在 8 位卡上，可能是 2（9）、3、4、5 或 7
在 16 位卡上，可能是 2（9）、3、4、5、7 或 10 至 15
确保这与系统上任何其他卡不同。请注意，就 Linux 而言，IRQ2 和 IRQ9 是相同的。你可以通过“cat /proc/interrupts”获取当前正在使用的一些中断列表。以下是 Vojtech Pavlik <vojtech@suse.cz> 提供的常见用途列表：

("不在总线上"意味着没有方法可以让卡生成此中断)

======   =========================================================
IRQ  0   定时器 0（不在总线上）
IRQ  1   键盘（不在总线上）
IRQ  2   中断控制器 2（不在总线上，也不会中断 CPU）
IRQ  3   COM2
IRQ  4   COM1
IRQ  5   空闲（如果你有 LPT2；有时是 COM3；可能是 PLIP）
IRQ  6   软盘控制器
IRQ  7   空闲（如果你不使用轮询驱动程序的 LPT1；PLIP）
IRQ  8   实时时钟中断（不在总线上）
IRQ  9   空闲（如果启用，VGA 垂直同步中断）
IRQ 10   空闲
IRQ 11   空闲
IRQ 12   空闲
IRQ 13   数字协处理器（不在总线上）
IRQ 14   固定磁盘控制器
IRQ 15   空闲（如果你有它，固定磁盘控制器 2）
======   =========================================================

.. note:: 

   IRQ 9 在某些视频卡上用于“垂直回扫”中断。此中断对于像视频游戏这样的应用来说非常方便，因为它恰好在每次屏幕刷新时发生一次，但不幸的是，IBM 从原始 VGA 开始取消了这个功能，因此许多 VGA/SVGA 卡并不支持它。由于这个原因，现代软件几乎都不使用此中断，如果您的视频卡支持它的话，几乎总是可以安全地禁用它。
如果由于某种原因，你的卡无法禁用此 IRQ（通常有一个跳线），一个解决方案是剪掉板子背面的印刷电路接触点：它是从左数第四个接触点。我不会对您尝试这种方法负责。
- Avery 的首选：IRQ2（实际上是 IRQ9）。但是要注意 VGA 的问题。
- 内存地址：与其他大多数卡不同，ARCnets 使用“共享内存”来复制缓冲区。确保它不与系统中的任何其他使用的内存冲突！

    ::

      A0000		- VGA 图形内存（如果你没有 VGA，则没关系）
      B0000		- 单色文本模式
      C0000		\ 这些之一是你的 VGA BIOS —— 通常是 C0000
      E0000		/
      F0000		- 系统 BIOS

任何小于 0xA0000 的地址都是一个糟糕的想法，因为它不在 640k 以上。
- Avery的最爱：0xD0000

  - 站点地址：每个ARCnet卡都有一个从0到255的“唯一”网络地址。与以太网不同，你可以通过跳线或开关（或者在某些卡上，通过特殊软件）自行设置这个地址。因为它只有8位，所以一个网络上只能有254张ARCnet卡。不要使用0或255，因为这些地址是预留的（虽然如果你确实使用它们可能会发生一些有趣的事情）。顺便说一句，如果你还没猜到的话，不要将此地址设置为与你的网络上的任何其他ARCnet卡相同！

- Avery的最爱：3和4。其实这并不重要。
- 可能会有ETS1和ETS2设置。这些设置在你的卡上可能有用也可能没用（许多手册称它们为“预留”），但用于更改在网络中启动计算机时使用的延迟。这仅在连接非常长距离的ARCnet网络时才必要，大约4公里左右；无论如何，在这里唯一真正的要求是网络上所有带有ETS1和ETS2跳线的卡都应将它们置于相同的位置。Chris Hindy <chrish@io.org> 发送了一个包含实际值的图表：

Carl de Billy <CARL@carainfo.com> 解释了J3和J4:

   J3示意图::

	   .-------
- 如果在总线拓扑中使用同轴电缆，必须移除J4跳线；将其放置在一个引脚上。
- 如果在使用双绞线的总线拓扑中，移动J3跳线，使它们连接中间引脚和最靠近RJ11连接器的引脚。同样，J4跳线必须被移除；为了存储，将它放在J4跳线的一个引脚上。
- 如果在使用双绞线的星型拓扑中，移动J3跳线，使它们连接中间引脚和最靠近RJ11连接器的引脚。

..

卡片顶部有另一组八个DIP开关。其中五个标记为MS0-MS4，似乎控制内存地址，另外三个标记为IO0-IO2，似乎控制卡片的基本I/O地址。

通过试错测试这很困难，且I/O地址顺序奇怪。这是通过设置DIP开关、重启电脑并尝试在不同地址加载ARCETHER（主要在0x200至0x400之间）来测试的。导致红色发送LED闪烁的地址，是我认为有效的地址。

此外，地址0x3D0似乎有特殊含义，因为ARCETHER数据包驱动程序可以正常加载，但没有红色LED闪烁。我不知道0x3D0是干什么用的。我建议使用0x300地址，因为Windows可能不喜欢低于0x300的地址。

```

内存开关设置了0x1000字节（0x100段单位或4K）的保留地址空间。例如，如果我设置地址为0xD000，它将占用从0xD000到0xD100的地址。

内存开关通过使用QEMM386隐秘启动并使用LOADHI查看哪些地址自动从高端内存区域排除，然后尝试使用这些地址加载ARCETHER来测试。
我建议使用 0xD000 作为 ARCnet 内存地址，并将 EMS 页面框架设置在 0xC000，同时使用 QEMM 隐身模式。这样，你几乎可以从 0xD100 开始获得连续的高端内存直到兆字节的末尾。

Memory Switch 0 (MS0) 在我的卡上设置为 OFF 时似乎不能正常工作。它可能在我的卡上有故障。首先尝试将其设置为 ON，如果不起作用，则将其设置为 OFF。（它可能是 0x200 位的一个修饰符？）

设置节点ID
^^^^^^^^^^^^^^^^^^^

SW2中的八个开关用于设置节点ID。网络上的每个节点必须有一个唯一的节点ID，且该ID不能为0。
开关1作为最低有效位（LSB），
将其中一个开关设置为关（Off）表示“1”，开（On）表示“0”。

节点ID是所有被设置为“1”的开关值的总和，
这些值如下：

   开关 | 值
   -------|-------
     1    |   1
     2    |   2
     3    |   4
     4    |   8
     5    |  16
     6    |  32
     7    |  64
     8    | 128

不要将此设置为0或255；这些值已被保留。

设置I/O基地址
^^^^^^^^^^^^^^^^^^^^^^^^^^^^

SW1中的开关1至6用于选择32个可能的I/O基地址之一，如下表所示：

	  | 十六进制
   开关 | 值
   -------|-------
     1    | 200
     2    | 100
     3    |  80
     4    |  40
     5    |  20
     6    |  10

I/O地址是所有被设置为“1”的开关值的总和。记住I/O地址空间在0x200以下的部分是为主板保留的，因此开关1应该总是被设置为关（Off）。

设置内存缓冲区（RAM）基地址
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

内存缓冲区（RAM）需要2K的空间。这个缓冲区的基地址可以位于十六个位置中的任何一个。然而，低于A0000的地址可能会导致系统挂起，因为那里有主RAM。
SW1中的跳线7-10选择内存基地址：

   开关          | 十六进制RAM
    7   8   9  10  | 地址
   ----------------|---------
   关  关  关  关 |  F0000（与主BIOS冲突）
   关  关  关  开  |  E0000
   关  关  开  关 |  D0000
   关  关  开  开  |  C0000（与视频BIOS冲突）
   关  开  关  关 |  B0000（与单色视频冲突）
   关  开  关  开  |  A0000（与图形冲突）

设置中断线路
^^^^^^^^^^^^^^^^^^^^^^^^^^

跳线块J1中的跳线1-5控制中断请求（IRQ）级别。开（ON）表示短路，关（OFF）表示断开：

    跳线              |  IRQ
    1   2   3   4   5   |
   ----------------------------
    开  关  关  关  关 |  7
    关  开  关  关  关 |  5
    关  关  开  关  关 |  4
    关  关  关  开  关 |  3
    关  关  关  关  开 |  2

未知的跳线与插座
^^^^^^^^^^^^^^^^^^^^^^^^^

对于这些我一无所知。我猜测J16和J17可能是超时跳线，而J18到J21中的一个可能用于选择ROM。此外，J6到J10以及J11到J15似乎将IRQ2到IRQ7连接到UFS上的某些引脚上。我无法猜测其用途。
