ARCnet 介绍
======================

ARCnet 是一种类似于流行的以太网但又在某些非常重要的方面有所不同的网络类型。首先，您可以获得至少两种速度的 ARCnet 卡：2.5 Mbps（比以太网慢）和 100 Mbps（比标准以太网快）。实际上还有其他类型，但它们不那么常见。据我所知，这些不同硬件类型是不兼容的，因此您不能将 100 Mbps 卡连接到 2.5 Mbps 卡等。据我所听到的消息，我的驱动程序确实可以支持 100 Mbps 卡，但我自己无法验证这一点，因为我只有 2.5 Mbps 的卡。它可能不会使您的 100 Mbps 卡达到饱和状态。别抱怨了。:)

您也不能将 ARCnet 卡连接到任何类型的以太网卡并期望其正常工作。ARCnet 有两种“类型”——星型拓扑和总线拓扑。这指的是卡应该如何连接在一起。根据大多数可用文档，只能将星型卡连接到星型卡，将总线型卡连接到总线型卡。这听起来有道理，对吧？其实并不完全正确；详见下面的“布线”。

一旦您克服了这些小障碍，ARCnet 实际上是一个设计得很好的标准。它使用了一种称为“修改后的令牌传递”的方法，这使其与所谓的“令牌环”卡完全不兼容，但使得数据传输比以太网更加可靠。事实上，ARCnet 会保证数据包安全到达目的地，即使不可能正确传送（例如因为电缆断裂或目标计算机不存在），它也会至少告知发送者。由于“令牌”的精心定义操作，它总会在最大时间内绕“环”一圈。这使得它适用于实时网络。此外，所有已知的 ARCnet 卡都有一个（几乎）相同的编程接口。这意味着用一个 ARCnet 驱动程序就可以支持任何卡，而以太网每个制造商有时会使用完全不同的编程接口，导致存在很多不同的、有时非常相似的以太网驱动程序。当然，始终使用相同的编程接口也意味着当像 PCI 总线主控 DMA 这样的高性能硬件设施出现时，很难利用它们。我们就不讨论这个了。然而，使 ARCnet 卡难以编程的一个因素是其数据包大小的限制；标准 ARCnet 只能发送长度最多为 508 字节的数据包。这小于互联网的“最低要求”576 字节，更不用说以太网的 MTU 1500 字节了。为了弥补这一不足，RFC1201 定义了一个额外的封装级别，我称之为“数据包拆分”，允许“虚拟数据包”增长到每个 64K，尽管通常保持在以太网风格的 1500 字节。有关 ARCnet 网络的优点和缺点（主要是优点）的更多信息，您可以访问“ARCnet 贸易协会”的 WWW 页面：

	http://www.arcnet.com

Avery 应 Vojtech 的请求对此进行了少许润色。经典 2.5 Mbps 版本的 ARCnet 可以通过两种不同类型的电缆连接：同轴电缆和双绞线。其他类型的 ARCnet 网络（100 Mbps TCNS 和 320 kbps - 32 Mbps ARCnet Plus）使用不同类型的电缆（Type1、光纤、C1、C4、C5）。对于同轴网络，您“应该”使用 93 欧姆的 RG-62 电缆。但是其他电缆也能很好地工作，因为 ARCnet 是一个非常稳定的网络。我个人使用的是 75 欧姆的电视天线电缆。
同轴电缆卡有两种不同的变体：用于总线（BUS）和星型（STAR）网络拓扑。它们基本上是一样的，唯一的区别在于安装的混合芯片不同。总线（BUS）卡使用高阻抗输出，而星型（STAR）卡使用低阻抗输出。低阻抗卡（STAR）在电气上等同于一个安装了终结器的高阻抗卡。

通常，ARCnet 网络是由星型（STAR）卡和集线器构建的。集线器有两种类型——主动式和被动式。被动式集线器是带有四个 BNC 连接器的小盒子，并且包含四个 47 欧姆的电阻：

```
	   |         | wires
	   R         + junction
	-R-+-R-      R 47 Ohm resistors
	   R
	   |
```

屏蔽层是连接在一起的。主动式集线器则复杂得多；它们需要供电，并且包含电子设备来放大信号并将其发送到网络的其他部分。它们通常有八个连接器。主动式集线器有两种变体——简单的和智能的。简单的变体仅进行信号放大，而智能变体则将所有通过的数据包解码为数字再重新编码回去。如果你在网络中有多个集线器，智能集线器会更好，因为多个简单的主动式集线器可能会降低信号质量。

现在我们来看看电缆连接的情况：

1. 一张卡到另一张卡。这是创建两台计算机网络最简单的方式。
2. 一张卡到一个被动式集线器。记住，集线器上未使用的连接器必须用 93 欧姆（或其他合适的阻值）的终结器正确终止。
（Avery 的注释：哦，我不知道这一点。我的电视电缆还是工作了。）
3. 一张卡到一个主动式集线器。这里不需要终止未使用的连接器，除非出于美观考虑。但是，在任何两台计算机之间不能超过十一个主动式集线器。当然，这并不限制网络中主动式集线器的数量。
4. 一个主动式集线器到另一个主动式集线器。
5. 一个主动式集线器到一个被动式集线器。
记住，你不能将两个被动式集线器连接在一起。这种连接造成的功率损耗太高，以至于网络无法可靠运行。

一个典型的 ARCnet 网络示例如下：

```
	   R                     S - 星型（STAR）类型卡
    S------H--------A-------S    R - 终结器
	   |        |            H - 集线器
	   |        |            A - 主动式集线器
	   |   S----H----S
	   S        |
		    |
		    S
```

总线（BUS）拓扑与以太网使用的非常相似。唯一的区别在于电缆和终结器：它们应该是 93 欧姆。以太网使用的是 50 欧姆阻抗。你使用 T 型连接器将计算机连接到一条电缆线上，即总线。你必须在电缆两端放置终结器。一个典型的总线（BUS）ARCnet 网络如下：

```
    RT----T------T------T------T------TR
     B    B      B      B      B      B

  B - 总线（BUS）类型卡
  R - 终结器
  T - T 型连接器
```

但这还不是全部！这两种类型的卡可以互相连接。根据官方文档，唯一的方法是使用主动式集线器：

```
	 A------T------T------TR
	 |      B      B      B
     S---H---S
	 |
	 S
```

官方文档还指出，你可以在总线（BUS）网络的两端使用星型（STAR）卡代替总线（BUS）卡和终结器：

```
     S------T------T------S
	    B      B
```

但根据我自己的实验，你可以在星型（STAR）拓扑网络中的电缆中间挂一个总线（BUS）类型卡。而且更进一步——如果你使用终结器，你可以用总线卡代替任何星型卡。这样你可以构建非常复杂的网络来满足各种需求！例如：

```
				  S
				  |
	   RT------T-------T------H------S
	    B      B       B      |
				  |       R
    S------A------T-------T-------A-------H------TR
	   |      B       B       |       |      B
	   |   S                 BT       |
	   |   |                  |  S----A-----S
    S------H---A----S             |       |
	   |   |      S------T----H---S   |
	   S   S             B    R       S
```

另一种完全不同的布线方案是双绞线（Twisted Pair）布线。每个双绞线卡有两个 RJ（电话线样式）连接器。这些卡通过一根电缆连接相邻的两张卡。两端用 RJ 93 欧姆终结器终止，终结器插在链末端卡的空连接器上。例如：

```
	  ___________   ___________
      _R_|_         _|_|_         _|_R_
     |     |       |     |       |     |
     |Card |       |Card |       |Card |
     |_____|       |_____|       |_____|
```

双绞线拓扑也有集线器。使用它们没有困难；你只需将双绞线链路连接到集线器的一端或两端即可。这样你可以创建几乎任何网络配置。在这里，任意两台计算机之间最多只能有 11 个集线器。例如：

```
    RP-------P--------P--------H-----P------P-----PR
			       |
      RP-----H--------P--------H-----P------PR
	     |                 |
	     PR                PR

    R - RJ 终结器
    P - 双绞线卡
    H - 双绞线集线器
```

像任何网络一样，ARCnet 也有有限的电缆长度。以下是最长的电缆长度，两端都是活动端点（活动端点可以是主动式集线器或星型卡）。
========== ======= ===========
RG-62       93 欧姆，最长可达 650 米  
RG-59/U     75 欧姆，最长可达 457 米  
RG-11/U     75 欧姆，最长可达 533 米  
IBM Type 1  150 欧姆，最长可达 200 米  
IBM Type 3  100 欧姆，最长可达 100 米  
========== ======= ==========

连接到被动集线器的所有电缆的最大长度对于 RG-62 电缆限制为 65 米；其他类型的电缆则更短。可以看出，在大型网络中使用被动集线器不是一个好主意。单个“BUS Trunk”的最大长度对于 RG-62 是约 300 米。网络中最远两点之间的最大距离限制为 3000 米。两个网卡/集线器之间的一段 TP 电缆的最大长度为 650 米。

设置跳线
===================

所有 ARCnet 网卡应该有总共四个或五个不同的设置：

- 输入/输出地址：这是您的 ARCnet 卡所在的“端口”。Linux ARCnet 驱动程序中探测的值仅从 0x200 到 0x3F0。（如果您的卡有额外的地址，请告诉我。）这不应该与系统中的任何其他设备相同。根据我从 Novell 获得的文档，MS Windows 偏好 0x300 或更高的值，否则会占用我的系统上的网络连接（至少）。我认为这可能是因为，如果您将卡设在 0x2E0 上，那么在 0x2E8 探测串行端口时可能会重置该卡，并且可能会严重地搞乱事情。
- Avery 的首选：0x300
- 中断请求（IRQ）：在 8 位卡上，可能是 2（9）、3、4、5 或 7；
  在 16 位卡上，可能是 2（9）、3、4、5、7 或 10-15。
确保这与系统中的任何其他卡不同。请注意，就 Linux 而言，IRQ2 和 IRQ9 是相同的。您可以使用 `cat /proc/interrupts` 获取当前使用情况的列表。以下是 Vojtech Pavlik <vojtech@suse.cz> 提供的一些常见用法：

```
("不在总线上"意味着没有方法可以让卡生成这个中断)

======   =========================================================
IRQ  0   定时器 0 （不在总线上）
IRQ  1   键盘（不在总线上）
IRQ  2   中断控制器 2（不在总线上，也不中断 CPU）
IRQ  3   COM2
IRQ  4   COM1
IRQ  5   空闲（如果您有 LPT2，则为 LPT2；有时为 COM3；可能是 PLIP）
IRQ  6   软盘控制器
IRQ  7   空闲（如果您不使用轮询驱动程序，则为 LPT1；PLIP）
IRQ  8   实时时钟中断（不在总线上）
IRQ  9   空闲（如果启用，则为 VGA 垂直同步中断）
IRQ 10   空闲
IRQ 11   空闲
IRQ 12   空闲
IRQ 13   数学协处理器（不在总线上）
IRQ 14   固定磁盘控制器
IRQ 15   空闲（如果您有第二个固定磁盘控制器，则为固定磁盘控制器 2）
======   =========================================================
```

.. note:: 

   在某些视频卡上，IRQ 9 用于“垂直回扫”中断。这种中断对视频游戏等应用非常有用，因为它恰好在每次屏幕刷新时发生一次。然而不幸的是，IBM 从最初的 VGA 开始取消了此功能，因此许多 VGA/SVGA 卡都不支持它。出于这个原因，现代软件几乎不使用这个中断，并且如果您的视频卡支持的话，几乎总是可以安全地禁用它。
如果您的卡由于某种原因无法禁用此 IRQ（通常有一个跳线），一种解决方案是剪断主板背面从左数第四个接触点的印刷电路接触点。如果尝试这样做，我不承担任何责任。
- Avery 的首选：IRQ2（实际上是 IRQ9）。注意 VGA。
- 内存地址：与其他大多数卡不同，ARCnets 使用“共享内存”来复制缓冲区。请确保它不与系统中的任何其他已使用的内存冲突！

```
A0000		- VGA 图形内存（如果您没有 VGA 可以使用）
B0000		- 单色文本模式
C0000		\ 这些之一是您的 VGA BIOS —— 通常是 C0000
E0000		/
F0000		- 系统 BIOS
```

任何小于 0xA0000 的地址都是一个坏主意，因为它不在 640K 以上。
- Avery 的最爱：0xD0000

  - 站点地址：每个 ARCnet 卡都有一个从 0 到 255 的“唯一”网络地址。与以太网不同，你可以通过跳线或开关（在某些卡上，通过特殊软件）自己设置这个地址。由于它只有 8 位，因此你只能在网络中使用 254 张 ARCnet 卡。不要使用 0 或 255，因为这些是保留的（虽然如果你确实使用了它们，可能会发生一些有趣的事情）。顺便说一句，如果你还没猜到的话，不要将此地址设置为与网络上的其他任何 ARCnet 相同！

  - Avery 的最爱：3 和 4。不过这并不重要。

  - 可能会有 ETS1 和 ETS2 设置。这些设置在你的卡上可能有用也可能没用（许多手册称其为“保留”），但用于更改网络计算机启动时使用的延迟。这仅在连接非常长距离的 ARCnet 网络（约 4 公里）时才必要；无论如何，这里唯一的真正要求是网络上所有带有 ETS1 和 ETS2 跳线的卡都必须将它们设置在相同的位置。Chris Hindy <chrish@io.org> 发来了一张表格，列出了这些值：

| ET1 | ET2 | 响应时间 | 重新配置时间 |
|-----|-----|----------|--------------|
| open | open | 74.7us | 840us       |
| open | closed | 283.4us | 1680us     |
| closed | open | 561.8us | 1680us     |
| closed | closed | 1118.6us | 1680us   |

确保你将网络上所有卡的 ETS1 和 ETS2 设置为相同的值。
此外，在许多卡上（虽然不是我的卡）有红绿 LED。Vojtech Pavlik <vojtech@suse.cz> 告诉我它们的意义如下：

| 绿色 | 红色 | 状态 |
|------|------|------|
| OFF  | OFF  | 电源关闭 |
| OFF  | 短闪 | 电缆问题（断开或未终止） |
| 短亮 | ON   | 卡初始化 |
| ON   | ON   | 正常状态——一切正常，没有活动 |
| ON   | 长闪 | 数据传输 |
| ON   | OFF  | 从不发生（可能是错误的 ID） |

跳线：

	在卡的底部靠近边缘连接器处有一个跳线阵列，标记为J1。它们控制IRQ和其他一些功能。只在IRQ引脚上放置一个跳线。
ETS1、ETS2用于非常长距离网络的时间设置。参见文件顶部的一般信息。
J2跳线位于两个引脚上。应该放置一个跳线，因为我拿到这张卡时已经有跳线了。我不知道这个跳线的作用是什么。
对于J3，有一个双跳线阵列。我不知道它的用途，但我拿到卡时已经有两个跳线了。这是一个六针网格，以两行三列的方式排列。跳线的配置如下：

	   .-------
o | o   o |
	   :-------:    ------> 连接器端在此方向 ------>
	 o | o   o |             在此方向有连接器 ------>
	   `-------'

在卡的顶部还有另一组八个DIP开关。其中有五个标记为MS0-MS4，似乎用于控制内存地址；另外三个标记为IO0-IO2，似乎用于控制卡的基本I/O地址。

通过试错测试这些I/O地址非常困难，并且它们的顺序也很奇怪。测试方法是设置DIP开关，重新启动计算机，并尝试在不同的地址加载ARCETHER（主要是在0x200到0x400之间）。导致红色发送LED闪烁的地址被认为是可以工作的。

此外，地址0x3D0似乎有特殊意义，因为ARCETHER数据包驱动程序可以正常加载，但没有红色LED闪烁。我不知道0x3D0是做什么用的。我建议使用0x300地址，因为Windows可能不喜欢低于0x300的地址。

设置节点 ID
^^^^^^^^^^^^

SW2 中的八个开关用于设置节点 ID。网络中的每个节点必须具有唯一的节点 ID，并且该 ID 必须不同于 0。开关 1（ID0）作为最低有效位（LSB）。节点 ID 是所有设置为“1”的开关值之和。这些值如下：

```

   =======  ======  =====

     2      ID1       2
     3      ID2       4
     4      ID3       8
     5      ID4      16
     6      ID5      32
     7      ID6      64
     8      ID7     128
   =======  ======  =====
```
