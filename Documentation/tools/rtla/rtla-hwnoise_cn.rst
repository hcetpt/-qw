============
rtla-hwnoise
============

------------------------------------------
检测并量化与硬件相关的噪声
------------------------------------------

:手册章节: 1

概要
========

**rtla hwnoise** [*选项*]

描述
===========

**rtla hwnoise** 收集了在 *interrupts disabled* 模式下运行的 *osnoise* 跟踪器的周期性摘要。通过禁用中断，以及因此导致的线程调度，仅允许非屏蔽中断和与硬件相关的噪声。
该工具还允许配置 *osnoise* 跟踪器以及收集跟踪输出。

示例
=======

在下面的例子中，**rtla hwnoise** 工具被设置为在包含8个核心/16个线程且启用了超线程的系统上的 *1-7* 号CPU上运行。
该工具被设置为检测任何高于 *一微秒* 的噪声，
运行 *十分钟* ，并在会话结束时显示报告摘要:

  # rtla hwnoise -c 1-7 -T 1 -d 10m -q
                                          与硬件相关的噪声
  持续时间:   0 00:10:00 | 时间单位为 μs
  CPU 周期         运行时间        噪声  % CPU可用   最大噪声   单次最大          硬件          NMI
    1 #599       599000000          138    99.99997           3            3           4           74
    2 #599       599000000           85    99.99998           3            3           4           75
    3 #599       599000000           86    99.99998           4            3           6           75
    4 #599       599000000           81    99.99998           4            4           2           75
    5 #599       599000000           85    99.99998           2            2           2           75
    6 #599       599000000           76    99.99998           2            2           0           75
    7 #599       599000000           77    99.99998           3            3           0           75

第一列显示 *CPU* 编号，第二列表明工具在会话期间运行了多少个 *周期* 。*运行时间* 是工具实际在CPU上运行的时间。*噪声* 列是工具观察到的所有噪声的总和，而 *% CPU可用* 是 *运行时间* 和 *噪声* 之间的关系。
*最大噪声* 列是工具在一个单个周期内检测到的最大硬件噪声，而 *单次最大* 是看到的最大单一噪声。
*硬件* 和 *NMI* 列显示由工具观察到的 *硬件* 和 *NMI* 噪声发生的总数。
例如，*CPU 3* 在 *599* 个周期内运行了 *1秒运行时间* 。整个执行过程中，CPU收到了 *86 μs* 的噪声，为应用留出了 *99.99997%* 的CPU时间。在最糟糕的一个周期里，CPU向应用程序造成了 *4 μs* 的噪声，但肯定是由多于一次的单一噪声造成的，因为 *单次最大* 噪声仅为 *3 μs* 。CPU存在 *硬件噪声*，每 *十分钟* 发生 *六次* 。CPU也有 *NMI*，频率更高：大约每 *秒七次* 。
理想情况下，该工具应该报告 *0* 与硬件相关的噪声。
例如，通过禁用超线程以消除硬件噪声，并禁用TSC监视器以消除NMI（这可以通过 **rtla hwnoise** 的跟踪选项来识别），在相同的硬件上实现了理想情况:

  # rtla hwnoise -c 1-7 -T 1 -d 10m -q
                                          与硬件相关的噪声
  持续时间:   0 00:10:00 | 时间单位为 μs
  CPU 周期         运行时间        噪声  % CPU可用   最大噪声   单次最大          硬件          NMI
    1 #599       599000000            0   100.00000           0            0           0            0
    2 #599       599000000            0   100.00000           0            0           0            0
    3 #599       599000000            0   100.00000           0            0           0            0
    4 #599       599000000            0   100.00000           0            0           0            0
    5 #599       599000000            0   100.00000           0            0           0            0
    6 #599       599000000            0   100.00000           0            0           0            0
    7 #599       599000000            0   100.00000           0            0           0            0
